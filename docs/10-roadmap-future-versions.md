# Future Roadmap

## Purpose

This document records intentionally deferred capabilities.

Features listed here are not required for POC v0.1.

The purpose is to preserve architectural direction while preventing scope creep during implementation.

---

# Versioning Philosophy

Version numbers represent meaningful capability increases rather than arbitrary releases.

Example:

v0.1
    ↓
v0.2
    ↓
v0.3
    ↓
v1.0

---

# v0.1 (POC)

Goal:

Deliver a fully functional specification-driven coding agent.

Capabilities:

- Textual TUI
- Specification Wizard
- Markdown + YAML Specs
- Repository Analysis
- Repository Knowledge Graph
- Incremental Graph Updates
- Context Assembly Engine
- Engineering Memory Pyramid
- DeepSeek Integration
- Provider Abstraction
- Human Approval Layer
- Read Tool
- Write Tool
- Edit Tool
- Bash Tool
- Test Execution
- Audit Logging
- Empty Repository Support

Success Criteria:

User can successfully implement a feature in a real repository using the complete workflow.
